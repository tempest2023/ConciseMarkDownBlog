# :star2: :star2: 贡献指南 :star2: :star2:

你想为这个项目做贡献！这太棒了。但在
这样做之前，请阅读以下关于如何贡献的简单步骤。这将
使工作更轻松，并避免在不被请求的事情上浪费时间。

## 在做之前讨论更改

- 首先，在仓库中打开一个问题，使用 [bug 跟踪器][1]，
  描述你想做的贡献、你发现的 bug 或任何
  其他想法。这将帮助我们让你正确地开始。

- 如果有意义，添加平台和软件信息（例如操作系统、
  Node.JS 版本等）、截图（以便我们能看到你看到的）。

- 建议在继续下一步之前等待反馈。
  但是，如果问题明确（例如拼写错误）且修复简单，你可以
  继续并修复它。

## 修复问题

- 在你的账户中 fork 项目并创建一个包含你修复的分支：
  `some-great-feature` 或 `some-issue-fix`。

- 在该分支中提交你的更改，按照我设置的
  eslint 代码风格编写代码。建议你添加一个测试。

- 如果项目包含 `package.json`，在 `contributors` 数组中添加你自己：

  ```json
  {
    "contributors": ["Your Name <and@email.address> (http://your.website)"]
  }
  ```

## 创建拉取请求

- 打开一个拉取请求，并在拉取请求消息中引用初始问题
  （例如 _fixes #<your-issue-number>_）。编写一个好的描述和
  标题，以便每个人都知道修复/改进了什么。

- 如果有意义，添加截图、gifs 等，以便更容易看到
  发生了什么。

## 等待反馈

在接受你的贡献之前，我们会审查它们。你可能会收到关于
你的修改代码中应该修复什么的反馈。如果是这样，只需继续在你的
分支中提交，拉取请求将自动更新。

## 皆大欢喜！

最后，你的贡献将被合并，每个人都会很高兴！:smile:
非常欢迎贡献！

谢谢！
